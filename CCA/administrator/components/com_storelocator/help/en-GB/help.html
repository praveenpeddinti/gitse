Joomla! Store Locator v1.8.2.0

Introduction

Place a Store Locator on your website using the power of Google Maps*. Allow your visitors to search for nearby locations and see them visually on a Map. You have the ability to specify unlimited Locations easily in the Joomla! Administration area. Your locations don't have to be stores, they can be anything you want. The Joomla Store Locator also provides directions via Google Maps. Visitors can search by any location that Google Maps can understand. This component is ideal to be used as a Dealer Locator, a Store Locator, an ATM / Branch Locator, a Member Locator or just about anything you can think of listing on a map in Joomla. Great for Stockists and other retail locations such as restaurants, transit systems, and points of interest.
* Your usage of the Google Maps API must meet the Google Maps License Requirements

For more information and the latest features please visit our website: www.sysgenmedia.com

Google Maps License Requirements

For-profit web sites are permitted to generate up to 25 000 map loads per day using the Google Maps JavaScript API v3. A map load is counted when a map is initialized on a web page. User interaction with a map after it has loaded (eg. panning the map, zooming the map, or changing from roadmap to satellite map) does not have any impact on usage limits.

In order to ensure that sites which experience short term spikes in usage are not adversely affected by the Maps API usage limits, only sites that exceed the limits for more than 90 consecutive days are subject to the limits.

If you operate a for-profit web site that consistently exceeds the Maps API usage limits every day, you will need to respond in one of the following ways:

  • Purchase additional quota through the Google APIs Console
  • Purchase a Maps API Premier license
  • Modify your site to reduce your usage to below the limits

* Note: Non-profit organizations are exempted from the Maps API usage limits, and are also encouraged to apply for a Google Earth Outeach Grant, which offers eligible non-profits a Maps API Premier license at no cost.

Installation and Setup

You no longer need a Google Maps API Key unless you meet the specific critiera above.
  • Navagate to Components → Store Locator → Location Manager from the drop-down menu on the back-end of your Joomla! installation
  • Set the Component Options/Parameters to your liking.
  • Add Your Location and Categoriess.
  • Add a new Menu Item to display the Store Locator on your site.
  • Optional: Enable the Search Module to search from an alternate page on your site.

Global Options

Click the Options button to open the Store Locator Configuration window. This window allows you to set default parameters for Locations.

Basic

  • Units: Set the units for search radius. Miles or Kilometers. (Default Miles)
  • Show Locations on Load: If set to Yes, locations are shown upon initial load. If set to No, then no locations will be shown on load. Total locations are still limited by Max Search Results. (Default Yes)
  • Marker Icon Preference: Determines which icons takes preference
  • jQuery Compatability Mode: Enable jQuery noConflict Mode. ONLY use if you are having trouble loading the store locator, and you use jQuery on your site. This MAY fix the problem. DO NOT USE UNLESS YOU ARE HAVING PROBLEMS AND USE JQUERY.

Map

  • Enable Directions: Adds a directions box to each marker popup. (Default Enabled)
  • Enable Zoom Wheel: Enable Zoom Control on the Map with the Mouse Wheel. (Default Enabled)
  • Enable Terrain: Enable the Terrain Map Type (Default Disabled)
  • Default Map Type: Set the default Map Type that is visible on load (Deafault Roads)
  • : Set the method of Clustering used on the map. (Default Google MarkerCluster)
  • Map Width: The width of the Map Area in pixels.
  • Map Height: The height of the Map Area in pixels.  
  • Hide List on Load: If set to hidden, hides the result list until a search is performed. Has no effect if the results list is set to disabled.
  • Initial Zoom State: If set to Auto, and the Locations are shown on load, the map will zoom to show all current locations. Otherwise it will use the settings below. (Default Auto)
  • Initial Zoom Level: The default zoom level of the Map Area. (Default 3)
  • Initial Center Latitude: The Default Center Latitude of the Map Area. (Default 40)
  • Initial Center Longitude: The Default Center Longitude of the Map Area. (Default -100)

Search

  • Enable Search: Enable the Front-End Search functionality. (Default Enabled)
  • Enable Category Search: Enable the ability to filter search results by Category. This is also limited by the available categories set in the Menu Item. (Default Enabled)
  • Enable Featured Search: Enable the ability to filter search results by Featured Items. (Default Enabled)
  • Enable Radius Search: Enable / Disable showing the radius search filter dropdown option. If hidden the default search radius will be used. (Default Enabled)
  • Enable Results List: Enable the list of locations to appear next to the Map. (Default Enabled)
  • Google Suggest: If the search term yields more then one possible location, offer suggestions to clarify the results.
  • Radius Allowed Options: A list of allowed values to be shown on the search radius dropdown. The list is independent of the units. The list must be Comma-Delimited.
  • Default Radius: Set the default Radius for the search dropdown. The value must be in the allowed values list above. (Default 25 mi/km)
  • Max Search Results: The maximum number of locations returned in search results. Must be greater than zero. (Default 250)
  • Max Search Zoom Level: The maximum zoom level when returning search results. This will prevent the map from zooming in too far when there are few search results. (Default 13)
  • Country Search Bias: Sets the geocoder to bias search results as if they were sent from the country specified. (Default: None)
  • Search Logging: Enable logging of search queries. (Default: No)
  • Search Limit: Enable limiting of searches per IP address. Requires Search Logging to be Enabled. Set to Zero to disable Limiting. (Default: 0)
  • Limiting Period: The length of time before a users search limit is reset. (Default: 1 Day)

Custom Fields

  • Custom Field Label 1 :The Label for the Custom Field. (Default: Custom Field 1)
  • Custom Field Label 2 :The Label for the Custom Field. (Default: Custom Field 2)
  • Custom Field Label 3 :The Label for the Custom Field. (Default: Custom Field 3)
  • Custom Field Label 4 :The Label for the Custom Field. (Default: Custom Field 4)
  • Custom Field Label 5 :The Label for the Custom Field. (Default: Custom Field 5)

Misc

  • Safe Uninstall: Setting this to yes will preserve your database tables on uninstall and r-install. (Default Yes)
  • Show Sysgen Media Link: If set to Yes, a small tagline will appear beneth the component with our copyright and link to our website. We appreciate the support!
  • Auto Detect Line Endings: Improves Compatability with CSV importing. Try to disable if you are having problems importing (Default Yes)
  • Include MooTools: Include MooTools Library from Joomla. Disable if you have conflicts. (Default Yes)
  • Include CSS: Include Default Style sheet. Adds a head tag to import the style sheet. (Default Yes)
  • Google Maps Premiere API Key: Optional: Only needed for high volume sites or Maps APi Premier Users.

Location Manager

How to Access

Select Components → Store Locator → Location Manager from the drop-down menu on the back-end of your Joomla! installation, or select the "Location Manager" link from the Store Locator - Categories .

Description

The Location Manager screen allows you to add location information to your Joomla! site. You can enter information such as name, address, Latitude and Longitude. Afterwards, you can use the Menu Manager to create front-end maps to the locations.

Column Headers

Click on the column heading to sort the list by that column's value.

  • ID. The ID number. This is a unique identification number for this item assigned automatically by Joomla!. It is used to identify the item internally, for example in internal links. You can not change this number.

  • Checkbox. Check this box to select one or more items. To select all items, check the box in the column heading. After one or more boxes are checked, click a toolbar button to take an action on the selected item or items. Many toolbar actions, such as Publish and Un-publish, can work with multiple items. Others, such as Edit, only work on one item at a time. If multiple items are checked and you press Edit, the first item will be opened for editing.

  • Name. The name of the Location. You can click on the Name to open the Location for editing.

  • Category. The Category that this Location belongs to. Clicking on the Category Title opens the Category for editing. See Category Manager - New/Edit. Note that Location Categories are separate from other Categories, such as those for Articles, Banners, Newsfeeds, and Web Links.

  • Address. The address of the Location.

  • Featured. Whether the item is a featured item. You can change the Featured state by clicking on the icon in this column.

  • Published. Whether the item has been published or not. You can change the Published state by clicking on the icon in this column.

  • Access Level. Who haspermissions to view this item.

Toolbar

At the top right you will see the toolbar:

The functions are:

  • Publish. To publish one or more items, select them and click on this button.

  • Un-publish. To un-publish one or more items, select them and click on this button.

  • Delete. To delete one or more items, select them and click this button. The selected items will be deleted.

  • Edit. Select one item and click on this button to open it in edit mode. If you have more than one item selected, the first item will be opened. You can also open an item for editing by clicking on its Name.

  • New. Click on this button to create a new item. You will enter the New page for this item.

  • Options. See Global Options section above.

  • Help. Opens this Help Screen.

CSV Import / Export

How to Access

Select Components → Store Locator → Import / Export from the drop-down menu on the back-end of your Joomla! installation, or select the "Import / Export" link from the Location Manager screen.

Description

The CSV Import / Export screen is where you can manage your location data via CSV files so that you can backup and save your existing location data or import a large data set of locations. With the new Batch Geocoder you can now import a large dataset and then look up coordinates for each location in a very quick and efficient manner. To use the Batch Geocoder, first Import your locations with the Lat/Lng fields set to 0. Then run the Batch Geocoder.

CSV Import

To import a list of locations browse for a CSV file on your computer. If the first row contains column headers check the box next to "First Row Contains Column Names" to skip importing the first row in the CSV file. Then click the "Import" button and wait untill the process is completed. You will be presented with the import results and let you know if there were any errors.

CSV Instructions

To see an example import file, add one location to the database from the location manager and then run an export.

Column Format:
Name,Address,Phone,Website,Lattitude,Longitude,Category,Email,Facebook,Twitter,Custom1,Custom2,Custom3,Custom4,Custom5,Tags,Featured,Published,"Start Publishing","Finished Publishing",Description

Example:
"Sysgen Media","81 Scudder Ave Northport NY","631-343-2211","http://www.sysgenmedia.com",40.899818,-73.347762,"General","test@example.com","http://facebook.com/SysgenMedia","sysgenmedia","custom text 1","custom text 2","custom text 3","custom text 4","custom text 5","Tag 1,Tag 2",1,1

CSV File - Syntax Rules:

  • File should be a Formatted as a CSV File.
  • Every Row MUST have 19 columns
  • Name, Category, Featured, and Published fields are required
  • Category can be specified by Name or by existing category ID. If the category doesnt exist it will be created. Category is Required.
  • Tags should be enclosed in quotes. Multiple tags should be delimited by commas. If the Tag doesn't exist, it will be created. Example: "Tag 1, Tag 2"
  • Featured should be specified as a 0 or 1. 0 = Unfeatured, 1 = Featured
  • Published should be specified as a 0 or 1. 0 - Unpublished, 1 = Published.
  • Start Publishing, Finished Publishing and Description are Optional Columns

If Latitude and Longitude are set to Zero you can use the Batch Geocoder to look up the correct coordinates.

CSV Export

To export a list of locations in a CSV file select the categories you wish to include in the export and click the "Export" button. The file will follow the same format as the CSV Import Requirements. For an example CSV Import file, add a location via the Location Manager and then perform a CSV Export.

Batch Geocoder

How to Access

Select Components → Store Locator → Batch Geocoder from the drop-down menu on the back-end of your Joomla! installation, or select the "Batch Geocoder" link from the Location Manager screen.

Description

Geocode multiple locations quiclky using the power of the Google Geocoding Web Service. The system will automatically locate any locations that have missing coordinates (locations with their coordinates set to Zero), upon clicking "Process Locations" the system will attempt to encode all your locations and automatically save their coordinates to the database. The system has a batch limit of 500 requests per run. If you need more, run it again and it will pick up where it left off. Please be patient, Geocoding is a time and resource intensive task and may take a few minutes to complete. In addition, Google imposes a daily limit of 2,500 requests per IP Address. Note: Because the service runs in the background in a very efficient manner the IP Address used is the Web Server's IP Address.

What is Geocoding?

Geocoding is the process of converting addresses (like "1600 Amphitheatre Parkway, Mountain View, CA") into geographic coordinates (like latitude 37.423021 and longitude -122.083739), which you can use to place markers or position the map. The Google Geocoding API provides a direct way to access a geocoder via an HTTP request. Additionally, the service allows you to perform the converse operation (turning coordinates into addresses); this process is known as "reverse geocoding."

Language Files

The Store Locator now includes multi-language support. We implemented the Joomla Language system as well as Joom!Fish support for Database information.

Joomla Language File

To modify any of the static text, the English language file provided will be located in:
Joomla 1.5: /language/en-GB/en-GB.com_storelocator.ini
Joomla 1.6/1.7: /components/com_storelocator/language/en-GB/en-GB.com_storelocator.ini

To add additional languages copy the ini file to the appropriate language folder and modify the entries. If you build a language file for a new language we would appricieate it if you could post it to our forums so we may include it in future releases.

Joom!Fish - For Joomla 1.5

We have also included Joom!Fish Content Element files so that yo can easily translate your Database entries for Category names and Location information. The content element files are automatically installed upon installation of Joom!Fish is detected. If you install Joom!Fish after you have already installed the store locator, you can install a Joom!Fish content element pack via the Joom!Fish installer. The content element pack is available as a separate download on the site. For more information visit: http://www.joomfish.net/
